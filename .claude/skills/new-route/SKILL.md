# Route Scaffolding

Use this skill when creating a new route under `/tree/$treeId/`. It covers the route file only — the page component it renders is a separate concern: see [design-system-standards](../design-system-standards/SKILL.md) for UI conventions and [typescript-standards](../typescript-standards/SKILL.md) for hooks, query keys, and naming.

## Route File

Location: `src/routes/tree/$treeId/<entity>.tsx`

```typescript
import { createFileRoute } from '@tanstack/react-router';
import { EntityPage } from '$/pages/EntityPage';

export const Route = createFileRoute('/tree/$treeId/<entity>')({
  component: function EntityRoute() {
    const { treeId } = Route.useParams();
    return <EntityPage treeId={treeId} />;
  },
});
```

For nested routes with a dynamic ID (e.g., `/tree/$treeId/entity/$entityId`):

- Create `src/routes/tree/$treeId/entity/$entityId/index.tsx` for the detail view
- Create `src/routes/tree/$treeId/entity/$entityId/edit.tsx` for the edit view

**Never edit `src/routeTree.gen.ts`** — it is auto-generated by `@tanstack/router-plugin`.

## Checklist

- [ ] Route file uses `createFileRoute()` with the correct path
- [ ] Route component reads params via `Route.useParams()` and passes them to the page component
- [ ] `routeTree.gen.ts` was NOT edited
