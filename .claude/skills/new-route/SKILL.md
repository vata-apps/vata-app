# Route & Page Scaffolding

Use this skill when creating a new route and page under `/tree/$treeId/`.

## Files to Create

| File                                   | Purpose          |
| -------------------------------------- | ---------------- |
| `src/routes/tree/$treeId/<entity>.tsx` | Route definition |
| `src/pages/<Entity>Page.tsx`           | Page component   |

## 1. Route File

Location: `src/routes/tree/$treeId/<entity>.tsx`

```typescript
import { createFileRoute } from '@tanstack/react-router';
import { EntityPage } from '$/pages/EntityPage';

export const Route = createFileRoute('/tree/$treeId/<entity>')({
  component: function EntityRoute() {
    const { treeId } = Route.useParams();
    return <EntityPage treeId={treeId} />;
  },
});
```

For nested routes with a dynamic ID (e.g., `/tree/$treeId/entity/$entityId`):

- Create `src/routes/tree/$treeId/entity/$entityId/index.tsx` for the detail view
- Create `src/routes/tree/$treeId/entity/$entityId/edit.tsx` for edit view

**Never edit `src/routeTree.gen.ts`** ÔÇö it is auto-generated by `@tanstack/router-plugin`.

## 2. Page Component

Location: `src/pages/<Entity>Page.tsx`. **Copy the shape of an existing page** ÔÇö `src/pages/IndividualsPage.tsx` is the canonical example (fully migrated to the current foundation). Invariants:

- Receives `{ treeId }` (list page) or `{ treeId, entityId }` (detail page); returns `JSX.Element`.
- Built from shared primitives in `src/components/ui/` (`Button`, `Typography`, `Icon`, ÔÇŽ) ÔÇö never raw HTML tags for interactive elements. Layout (flex/grid, spacing) is a page-level `<page>.css.ts` styled from `vars.*` (see `src/pages/list-page.css.ts`), not JSX layout-prop components. See [design-system-standards](../design-system-standards/SKILL.md) for the full reuse/compose/new-primitive decision tree.
- All user-facing text via `useTranslation()` ÔÇö never hardcode strings (project i18n rule).
- Data via TanStack Query hooks in `src/hooks/use<Entity>.ts`; handle `isLoading` / `isError`.

## 3. Conventions

Path aliases, the query-key factory, `staleTime`, and naming are cross-cutting rules owned by [typescript-standards](../typescript-standards/SKILL.md) ÔÇö follow it (the checklist below chains to it). Route/page-specific notes only:

### Modal dialogs (create/edit forms)

Create/edit forms open as `$components/ui/dialog` modals ÔÇö see [Layouts ┬ž Forms Open as Modals](../../../docs/ui/layouts.md) for the mechanism. Follow `src/components/individuals/person-editor-dialog.tsx`, not `src/components/trees/new-tree-modal.tsx` (an older, pre-migration modal still on `@radix-ui/themes`'s `Dialog`).

### Error handling

Surface `isError` from queries in the UI, and show mutation errors via an i18n message (never a hardcoded English string). See `IndividualsPage.tsx` for the pattern.

## Checklist

Before finishing:

- [ ] Route file uses `createFileRoute()` with correct path
- [ ] Page component receives `treeId` (and `entityId` for detail views)
- [ ] `routeTree.gen.ts` was NOT edited
- [ ] Page uses `src/components/ui/` primitives + a page-level `.css.ts` for layout + `useTranslation()` (no raw HTML layout, no hardcoded strings)
- [ ] Modal is a `$components/ui/dialog` organism composed via `open` / `onOpenChange` (no hand-rolled a11y)
- [ ] Query key added to `$/lib/query-keys` if new entity
- [ ] Also apply the [typescript-standards](../typescript-standards/SKILL.md) checklist
